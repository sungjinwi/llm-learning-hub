# Reverse Engineering

## 목적

완성된 코드나 낯선 코드를 위에서 아래로 해체해 구조를 이해한다.

## 사용할 때

- 긴 코드가 부담스러울 때
- 라이브러리 예제가 무슨 흐름인지 모를 때
- 모델 학습 코드의 데이터 흐름을 파악해야 할 때

## 분석 순서

1. 입력과 출력 찾기
2. 데이터 shape 추적
3. 주요 함수와 클래스의 역할 분리
4. 코드 블록을 3~5개 논리 단위로 나누기
5. 각 블록의 실패 가능성 찾기
6. 변수명을 의미 있게 바꿔보기

## 산출물

- 데이터 흐름 요약
- 핵심 변수 표
- 위험 지점
- 리팩토링 후보
