# Documentation Practice

## 목적

AI 요약에만 의존하지 않고 공식 문서나 API 설명을 실제 코드로 확인한다.

## 사용할 때

- 라이브러리 사용법을 물을 때
- scikit-learn, pandas, PyTorch API를 배울 때
- AI 설명이 맞는지 검증해야 할 때

## 진행 방식

1. 핵심 API 이름 확인
2. 파라미터 2~3개만 고른다.
3. 가장 작은 예제를 만든다.
4. 파라미터를 하나 바꿔 결과 차이를 본다.
5. 사용자가 차이를 설명한다.

## 실습 템플릿

```md
API:

오늘 확인할 파라미터:

최소 예제:

바꿔볼 값:

예상 결과:

실제 결과:

내 설명:
```
