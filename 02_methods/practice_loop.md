# Practice Loop

보고서의 핵심은 이론 설명을 듣는 것이 아니라, AI와 상호작용하며 직접 인출하고 구현하는 것이다. 모든 학습 대화는 가능한 한 아래 루프를 따른다.

## 기본 루프

1. 목표 확인
2. 사전 질문
3. 최소 개념 설명
4. 코드 실습
5. 실행 전 예측
6. 답변 또는 코드 작성
7. 채점과 피드백
8. 변형 문제
9. 약점 기록
10. 복습 예약

## 코드 실습 유형

### 1. 빈칸 코드

사용자가 핵심 문법이나 API를 직접 채운다.

```python
numbers = [1, 2, 3, 4]
result = [___ for n in numbers if ___]
print(result)
```

### 2. 출력 예측

실행 전에 결과를 말하게 한다.

```python
items = {"a": 1, "b": 2}
items["c"] = items["a"] + items["b"]
print(items)
```

### 3. 오류 찾기

일부러 잘못된 코드를 주고 원인을 찾게 한다.

```python
values = [10, 20, 30]
print(values[3])
```

### 4. 작은 구현

5~20줄 안에서 직접 구현하게 한다.

예:

- 리스트에서 짝수만 골라 합계 구하기
- DataFrame에서 결측치가 있는 행 찾기
- train/test split 후 accuracy 계산하기
- tensor shape을 바꾸고 결과 설명하기

### 5. 역공학

완성 코드를 보고 다음을 설명하게 한다.

- 입력
- 중간 처리
- 출력
- 핵심 변수
- 실패 가능성

## AI의 제한

AI는 사용자가 시도하기 전까지 전체 코드를 먼저 완성하지 않는다.

대신 다음을 제공한다.

- 목표
- 입력 예시
- 출력 예시
- 빈 코드 골격
- 힌트

## 완료 기준

실습은 사용자가 다음 중 하나를 해내야 완료된다.

- 코드 작성
- 출력 예측
- 오류 원인 설명
- 데이터 흐름 설명
- 같은 개념의 변형 문제 풀이
