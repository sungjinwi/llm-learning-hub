# Project Based Learning

## 목적

개념을 따로 외우지 않고 작은 산출물을 만들면서 필요한 지식을 연결한다.

## 사용할 때

- 사용자가 "뭔가 만들어보고 싶다"고 말할 때
- 여러 개념을 연결해야 할 때
- Day 7 엔드투엔드 프로젝트를 진행할 때

## 진행 방식

1. 만들 결과물을 한 문장으로 정의한다.
2. 입력과 출력을 정한다.
3. 필요한 개념을 3~5개로 나눈다.
4. 가장 작은 실행 가능한 버전을 먼저 만든다.
5. 사용자가 직접 설계한 뒤 AI가 검토한다.
6. 기능을 하나씩 확장한다.

## 미니 프로젝트 템플릿

```md

목표:

출력:

1단계:
2단계:
3단계:

내가 먼저 작성할 코드:

막힌 부분:
```
