# Prompt Quality Patterns

이 문서는 이 학습 시스템에서 재사용할 고품질 prompt 패턴을 정리한다.

핵심 원칙은 사용자가 매번 복잡한 prompt를 직접 쓰게 만드는 것이 아니다. 사용자는 짧고 자연스럽게 말하고, AI가 필요한 경우 내부적으로 구조화해서 처리한다.

## 1. 기본 구조화 패턴

복잡한 요청은 다음 구조로 해석한다.

```md
목표:
맥락:
입력:
제약:
출력 형식:
검증 기준:
불확실할 때 처리:
```

사용자가 이렇게 길게 쓰지 않아도 된다. AI가 사용자의 자연어 요청에서 위 항목을 추출하고, 부족한 항목만 필요할 때 질문한다.

## 2. XML 또는 delimiter 사용 기준

XML 태그는 사용자가 반드시 직접 써야 하는 형식이 아니다.

사용 기준:

- 긴 문서와 지시문이 섞일 때
- 예시와 실제 입력을 구분해야 할 때
- RAG에서 근거 문서와 질문을 분리해야 할 때
- 출력 형식을 엄격히 지켜야 할 때

예:

```md
<instructions>
아래 문서에 근거해서만 답하라.
근거가 없으면 "문서에서 확인할 수 없습니다"라고 답하라.
</instructions>

<context>
문서 내용...
</context>

<question>
사용자 질문...
</question>

<output_format>
- 답변:
- 근거:
- 확신도:
</output_format>
```

## 3. 학습 튜터 패턴

```md
목표:
사용자가 개념을 직접 인출하게 한다.

규칙:
- 정답을 먼저 보여주지 않는다.
- 먼저 1-3개 질문을 낸다.
- 사용자의 답변 뒤에 부족한 부분만 설명한다.
- 반드시 작은 코드 실습이나 출력 예측으로 연결한다.
- 틀린 내용은 weak_points 형식으로 정리한다.
```

## 4. 공식 문서 확인 패턴

```md
목표:
공식 문서 기준으로 정확한 사용법을 확인한다.

절차:
1. 공식 문서에서 현재 API 또는 개념을 확인한다.
2. 문서 기준 사실과 AI의 추론을 구분한다.
3. 가장 작은 예제를 만든다.
4. 사용자가 실행 전 출력 또는 데이터 흐름을 예측하게 한다.
5. 참고한 문서를 학습 로그에 남긴다.
```

## 5. RAG 답변 패턴

```md
<instructions>
아래 context에 있는 정보만 근거로 답하라.
근거가 부족하면 "문서에서 확인할 수 없습니다"라고 답하라.
답변에는 사용한 source를 함께 표시하라.
</instructions>

<context>
검색된 chunk 목록
</context>

<question>
사용자 질문
</question>

<output_format>
답변:
근거 source:
근거 문장:
확인 불가 항목:
</output_format>
```

## 6. 코드 리뷰와 디버깅 패턴

```md
목표:
코드의 입력, 처리, 출력, 실패 가능성을 분리해서 이해한다.

분석 순서:
1. 입력 데이터와 타입
2. 중간 데이터 구조
3. 출력
4. 실패 가능한 줄
5. 최소 재현 코드
6. 수정 전 예상 결과
```

## 7. Prompt 개선 체크리스트

- 목표가 한 문장으로 분명한가?
- 입력과 맥락이 충분한가?
- 출력 형식이 명확한가?
- 예시가 필요한데 빠져 있지 않은가?
- 서로 충돌하는 지시가 없는가?
- 한 번에 너무 많은 일을 시키고 있지 않은가?
- 근거가 필요한 작업인데 context 없이 답하게 하고 있지 않은가?
- JSON을 요구하면서 schema를 주지 않은 것은 아닌가?
- 최신 API 사용법인데 공식 문서를 확인하지 않은 것은 아닌가?

## 8. 피해야 할 패턴

- "최대한 잘해줘"처럼 성공 기준이 없는 요청
- "완벽하게 해줘"처럼 검증 기준이 없는 요청
- 정답이 문서에 있어야 하는데 문서를 주지 않는 요청
- 출력 형식을 요구하면서 예시나 schema를 주지 않는 요청
- 최신 API 사용법을 기억만으로 단정하는 요청
- 여러 독립 작업을 한 번에 묶은 요청
- reasoning model에 무조건 "think step by step"을 붙이는 방식

## 9. 사용자가 요청할 수 있는 짧은 명령

```text
이 질문 구조화해줘
이 prompt 개선해줘
RAG용 prompt로 바꿔줘
공식 문서 확인용 prompt로 바꿔줘
학습 튜터용 prompt로 바꿔줘
```

AI는 사용자의 원문을 바탕으로 구조화된 prompt를 제안하되, 사용자가 다시 복사해서 질문하게 만들기보다 가능한 경우 바로 그 구조를 적용해 답변한다.
