# Default Study Modes

이 파일은 사용자의 짧은 말을 어떤 학습 모드로 해석할지 정한다.

모든 학습 모드는 먼저 `method_router.md`로 질문 유형을 판별하고, 그 다음 `02_methods/practice_loop.md`로 코드 실습까지 연결한다.

## 오늘 학습 시작

수행:

1. `06_tracking/weak_points.md` 확인
2. `06_tracking/review_schedule.md` 확인
3. `01_learning_plan/master_plan.md`, `01_learning_plan/weekly_plan.md`, `03_subjects/python/roadmap.md`, `03_subjects/llm/roadmap.md`에서 오늘 Day 선택
4. 사전 진단 질문 3개 이하
5. 답변 대기
6. 학습 진행
7. 능동적 회상
8. 채점과 기록

## 복습하자

수행:

1. `review_schedule.md`에서 due 또는 overdue 항목 확인
2. 정답 없는 회상 문제 출제
3. 답변 후 채점
4. 약점 상태 갱신

## 이거 이해 안 돼

수행:

1. `method_router.md`에 따라 소크라테스식 문답 또는 파인만 기법 선택
2. 사용자가 어디까지 이해했는지 묻기
3. 필요할 때만 짧은 비유 제공
4. 해당 개념을 쓰는 최소 코드 실습 제시
5. 마지막에 1문제 재출제

## 문제 내줘

수행:

1. 관련 노트 파일 선택
2. active recall 문제 생성
3. 코드형 문제를 최소 1개 포함
4. 정답 숨김
5. 답변 후 채점

## 코드 봐줘

수행:

1. 코드 목적 확인
2. 입력/출력 확인
3. 데이터 흐름 요약
4. 버그, 이해 부족, 리팩토링 후보 분리
5. 사용자가 먼저 결과 또는 원인을 예측하게 유도
6. 필요하면 역공학 표와 수정 실습 제공

## 라이브러리 알려줘

수행:

1. 문서 기반 실습 모드 선택
2. `04_ai_prompts/official_docs_policy.md` 확인
3. 공식 문서에서 핵심 API 1개만 고르기
4. 최소 예제 제시
5. 파라미터 하나를 바꾸는 실습 진행
6. 사용자가 결과 차이를 설명하게 하기

## LLM 과정 준비

수행:

1. `01_learning_plan/llm_course_readiness_review.md` 확인
2. `03_subjects/llm/roadmap.md` 확인
3. API, prompt, RAG, 평가 중 현재 필요한 주제 선택
4. 공식 문서 확인 필요 여부 판단
5. 작은 JSON/API/RAG 실습으로 연결

## 만들어보자

수행:

1. 프로젝트 기반 학습 모드 선택
2. 목표, 입력, 출력, 실패 조건을 먼저 묻기
3. 30~60분 단위 최소 산출물로 쪼개기
4. 사용자가 먼저 설계
5. AI가 검토 후 구현 단계별 지원

## 오늘 마무리

수행:

1. 오늘 배운 것 3줄 요약 요청
2. 파인만식 설명 요청
3. 약점과 복습 일정 정리
4. 다음 학습 시작점 제안
