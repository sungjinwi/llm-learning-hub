# Day 4 LLM Theory Lite - API

## 적용 학습법

- 기본: 개념 연결 회상
- 보조: API 응답 역공학

## 목표

API 호출을 배우기 전에 LLM이 텍스트를 어떻게 다루고 왜 비용, context window, 출력 안정성이 중요한지 이해한다.

## 실습 1 - token과 비용 감각

아래 두 요청 중 어떤 요청이 더 많은 token을 사용할 가능성이 큰지 설명한다.

```text
요청 A:
Python dict를 한 문장으로 설명해줘.

요청 B:
아래 20페이지 문서를 모두 읽고 핵심 내용을 요약한 뒤, 관련 코드 예제와 테스트까지 작성해줘.
```

질문:

1. 어떤 요청이 더 비용이 커질 가능성이 높은가?
2. 비용은 왜 글자 수가 아니라 token 수와 관련되는가?
3. 긴 문서를 매번 API에 넣는 방식의 문제는 무엇인가?

## 실습 2 - context window

```text
상황:
사용자 질문은 마지막에 있지만, 앞부분에 긴 문서 50개가 함께 들어간다.
```

질문:

1. context window가 제한되어 있으면 어떤 문제가 생길 수 있는가?
2. 중요한 정보가 너무 뒤나 너무 앞에 있을 때 모델 답변에 어떤 영향이 있을 수 있는가?
3. RAG가 이 문제를 어떻게 줄여주는가?

## 실습 3 - next-token prediction

다음 설명을 네 말로 고쳐라.

```text
LLM은 질문을 받으면 인터넷에서 정답을 검색해서 가져온다.
```

질문:

1. 위 설명에서 틀린 부분은 무엇인가?
2. next-token prediction 관점에서 더 정확히 설명하면 어떻게 되는가?
3. 그래서 공식 문서 확인이나 RAG가 필요한 이유는 무엇인가?

## 완료 기준

- token, context window, next-token prediction을 API 사용 이유와 연결해 설명할 수 있다.
- LLM이 자연스럽게 답한다고 해서 사실 검증이 끝난 것이 아님을 설명할 수 있다.
