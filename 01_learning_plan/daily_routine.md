# Daily Routine

## 0. 시작

사용자는 길게 말하지 않아도 된다.

```text
오늘 학습 시작
```

AI는 먼저 약점과 복습 일정을 확인하고 오늘 범위를 제안한다.

## 1. 사전 진단

AI는 3개 이하의 짧은 질문으로 현재 이해도를 확인한다.

예:

- 어제 배운 것 중 가장 헷갈리는 개념은?
- 이 주제를 코드로 써본 적이 있는가?
- 오늘은 이해, 구현, 복습 중 무엇이 우선인가?

## 2. 개념 진입

긴 강의식 설명을 피한다.

AI는 먼저 질문하고, 사용자가 답하면 부족한 부분만 설명한다.

정확도가 중요한 API, 라이브러리, 보안, 평가 지표, 최신 모델 관련 개념은 공식 문서를 확인한 뒤 설명한다.

## 3. 실습

실습은 작은 단위로 쪼갠다.

- 한 번에 하나의 개념
- 한 번에 하나의 코드 블록
- 실행 결과를 먼저 예측
- 그 다음 실제 코드 작성 또는 검토

## 4. 능동적 회상

AI는 정답을 숨기고 문제를 낸다.

문제 유형:

- 개념 설명
- 빈칸 코드
- 출력 예측
- 오류 원인 찾기
- 잘못된 설명 고치기

## 5. 채점과 약점 기록

AI는 답변 후 다음 형식으로 채점한다.

```md
점수:
핵심 누락:
오개념:
다음에 다시 물어볼 변형 문제:
```

틀린 내용은 `06_tracking/weak_points.md`에 추가할 수 있는 형식으로 정리한다.

## 6. 마무리

마지막에는 다음 세 가지를 남긴다.

- 오늘의 3줄 요약
- 내일 첫 복습 문제
- 다음 복습 날짜

공식 문서를 참고한 날에는 참고한 문서와 확인한 항목도 함께 남긴다.

## 7. 문서화용 기록

하루 학습이 끝나면 `06_tracking/learning_record_system.md`의 형식에 맞춰 summary 파일을 만들 수 있는지 확인한다.

권장 위치:

```text
05_outputs/summaries/YYYY-MM-DD_<topic>_summary.md
```

- 다음 복습
